# 内容监控工具 - 技术方案

## 项目概述
- **项目名称**: 内容监控中心
- **项目类型**: Next.js 前端原型
- **核心功能**: 多平台内容监控、选题分析与报告生成

## 整体结构

### 1. 监控分类管理（顶层）
支持创建和管理多个监控分类，例如：
- Claude Code 选题监控
- Vibe Coding 选题监控

每个分类相互独立，拥有各自的监控配置和内容数据。

### 2. 分类内部 Tab 页
每个分类下包含三个 Tab：
- Tab 1：内容
- Tab 2：选题分析与报告
- Tab 3：监控设置

## 各 Tab 详细需求

### Tab 1：内容
- **平台筛选**: 平铺标签/按钮形式，用户可直接点选切换平台
- **时间线浏览**: 日期卡片横向滚动 + 数据标记
- **假数据**: 预先填充，方便预览和调整

### Tab 2：选题分析与报告
- **核心功能**: AI 分析热门内容，产出分析报告和选题洞察
- **时间选择**: 日期列表带预览摘要，可直接点选的日期卡片
- **选题维度**: 支持筛选查看一段时间内的选题汇总

**选题内容字段**:
- 方向标题
- 选题简介（包括为什么做这个选题、爆点及增长空间）

### Tab 3：监控设置
- 监控平台（抖音、小红书、微博、B站等）
- 对标关键词
- 对标博主/账号
- 运行设置（每天定时运行）

## 设计要求

### 设计风格
- **风格**: 简洁专业，适合内容运营人员日常使用
- **主色调**: 蓝色系 (#3b82f6)
- **字体**: 系统字体栈

### 交互优化
1. **平台筛选**: 平铺标签按钮，避免下拉选择
2. **日期选择**: 日期卡片横向滚动，带数据量标记
3. **报告选择**: 日期卡片带报告标题预览
