## 1. 系统简介

内容工作台是一款面向微信公众号运营者的全流程内容创作管理平台，核心能力包括：

- **📥 文章采集**：采集微信公众号文章，建立专属素材库
- **🔍 公众号采集**：搜索和订阅公众号，自动监控新文章
- **🔥 热门选题**：多源聚合热点话题，AI推荐选题方向
- **📊 选题分析**：AI评估选题潜力，提升爆款命中率
- **✍️ AI创作**：6步SOP流程，从选题到成文一键完成
- **📋 待发布管理**：草稿审核与发布管理
- **📈 数据分析**：文章数据统计与趋势分析
- **🔄 闭环优化**：基于发布效果持续优化内容策略

## 2. 环境准备与安装

### 2.1 系统要求

| 项目 | 要求 |
|------|------|
| 操作系统 | macOS / Linux / Windows |
| Node.js | 18.x 及以上 |
| PostgreSQL | 14.x 及以上 |
| 内存 | 建议 4GB 以上 |
| 磁盘 | 建议 10GB 以上可用空间 |

### 2.2 安装步骤

**方式一：本地开发环境启动**

```bash
# 1. 进入项目目录
cd content-monitor

# 2. 安装依赖
npm install

# 3. 确保 PostgreSQL 已启动
pg_isready -h localhost -p 5432
# 输出 "accepting connections" 表示数据库已就绪

# 4. 推送数据库 schema（首次运行或数据库结构更新后执行）
npx drizzle-kit push

# 5. 配置环境变量
cp .env.local.example .env.local
# 编辑 .env.local 填入实际配置（详见 2.3 节）

# 6. 启动开发服务器
npm run dev

# 7. 浏览器访问
# http://localhost:3003
```

**方式二：Docker Compose 启动（推荐）**

```bash
# 1. 进入项目目录
cd content-monitor

# 2. 配置环境变量
cp .env.local.example .env.local
# 编辑 .env.local 填入实际配置

# 3. 一键启动
docker-compose up -d

# 4. 浏览器访问
# http://localhost:3003
```

### 2.3 环境变量配置说明

创建 `.env.local` 文件，按需配置以下变量：

```bash
# ===== 必填 =====
# 数据库连接地址
DATABASE_URL=postgresql://content_monitor:your_password@localhost:5432/content_monitor_db

# ===== 微信公众号发布功能（如需发布到微信则必填）=====
WECHAT_APPID=your_wechat_appid
WECHAT_SECRET=your_wechat_secret
WECHAT_AUTHOR=默认作者名称

# ===== AI 创作功能（至少配置一个 LLM 服务商）=====
# 在系统设置 > API配置 中配置，无需在此填写

# ===== 搜索服务（热门选题功能使用，按需配置）=====
TAVILY_API_KEY=your_tavily_api_key
TIANGONG_API_KEY=your_tiangong_api_key
MINIMAX_API_KEY=your_minimax_api_key
MINIMAX_GROUP_ID=your_minimax_group_id

# ===== 图片搜索（创作配图使用，按需配置）=====
UNSPLASH_ACCESS_KEY=your_unsplash_access_key
PEXELS_API_KEY=your_pexels_api_key
EXA_API_KEY=your_exa_api_key
```

> **提示**：即使不配置搜索和图片 API Key，系统仍可正常使用，部分功能会降级为免费搜索引擎（维基百科、Bing、DuckDuckGo 等）。

## 3. 首次启动与登录

### 3.1 注册账号

1. 打开浏览器访问 `http://localhost:3003`
2. 进入登录页面，点击 **"注册账号"** 切换到注册模式
3. 填写注册信息：
   - **用户名**：用于登录的账号名
   - **邮箱**：用于找回密码
   - **密码**：至少6位
   - **显示名称**：在系统中显示的昵称
4. 点击 **"注册"** 按钮

### 3.2 登录系统

1. 在登录页面输入 **用户名** 和 **密码**
2. 点击 **"登录"** 按钮
3. 登录成功后自动跳转到首页仪表盘

> **默认管理员账号**：首次部署后数据库中已创建管理员账号 `admin`，邮箱 `admin@example.com`。

## 4. 系统设置（必做）

登录后，首先需要完成以下基础配置，否则核心功能无法正常使用。

### 4.1 配置 LLM 大模型（必做）

AI 创作功能依赖大模型，必须先配置至少一个 LLM 服务商。

1. 点击左侧菜单 **⚙️ 系统设置**
2. 切换到 **🤖 API 配置** 标签
3. 配置大模型：
   - **服务商**：选择 AI 服务商
     - OpenAI（GPT-4o 等）
     - Anthropic Claude（Claude 3.5 等）
     - DeepSeek（DeepSeek Chat 等）
     - Kimi / 月之暗面（Moonshot V1 等）
     - MiniMax（MiniMax-M2.7 等）
     - 智谱AI（GLM-4 等）
   - **API Key**：输入对应服务商的 API 密钥
   - **模型**：选择要使用的具体模型
   - **API URL**（部分服务商需要）：填写自定义 API 地址
4. 点击 **"保存配置"**

> **安全提示**：API Key 在前端仅显示掩码（如 `sk-****`），不会明文展示。更换密钥时输入新密钥即可，留空则保留原配置。

### 4.2 配置公众号账号（如需发布到微信）

1. 在系统设置中切换到 **📝 公众号配置** 标签
2. 点击 **"+ 添加账号"**
3. 填写公众号信息：
   - **账号名称**（必填）：公众号名称
   - **AppID**：微信公众号后台获取
   - **AppSecret**：微信公众号后台获取
   - **作者名称**：默认署名
   - **目标用户群体**：描述目标用户（用于AI创作时参考）
   - **读者画像**：详细描述读者特征
   - **内容风格**：选择内容风格（专业深度/轻松幽默/干货实用等）
   - **主要话题领域**：用逗号分隔，如 `科技,AI,创业`
   - **语言风格偏好**：选择语言风格
4. 点击 **"保存"**
5. 可设置某个账号为 **默认账号**（点击"设为默认"按钮）

### 4.3 配置图片生成（可选）

1. 在 API 配置标签中，找到 **🖼️ 图片生成配置**
2. 勾选 **"AI 生成配图（MiniMax 文生图）"**
3. 系统将使用 MiniMax AI 自动生成与文章内容相关的配图

### 4.4 菜单管理（可选）

1. 切换到 **📋 菜单管理** 标签
2. 可以开启或关闭侧栏菜单项的显示
3. 修改后点击 **"💾 保存设置"**

### 4.5 Prompt 管理（高级用户可选）

1. 切换到 **📝 Prompt管理** 标签
2. 可查看和编辑 AI 创作使用的 Prompt 模板
3. 点击 **"编辑"** 修改 Prompt 内容
4. 点击 **"保存"** 应用修改

## 5. 核心功能使用流程

### 5.1 文章采集

文章采集功能用于从微信公众号采集文章，建立素材库。

**操作步骤：**

1. 点击左侧菜单 **📥 文章采集**
2. 采集模式操作：
   - **输入文章链接**：在输入框中粘贴微信公众号文章 URL
   - **批量采集**：支持一次输入多个链接（每行一个）
3. 点击 **"采集"** 按钮
4. 系统自动提取文章内容，包括：
   - 文章标题、作者、发布时间
   - 正文内容（自动清理广告和排版）
   - 封面图片
   - 阅读量、点赞数等互动数据
5. 采集完成后，文章进入 **文章库**
6. 可对已采集文章进行：
   - **查看**：查看文章详情
   - **收藏**：标记为收藏
   - **标签**：添加自定义标签
   - **导出**：导出为 Markdown / PDF 格式
   - **删除**：移除文章

### 5.2 公众号采集

公众号采集功能用于搜索和订阅公众号，自动监控新文章发布。

**操作步骤：**

1. 点击左侧菜单 **🔍 公众号采集**
2. **搜索公众号**：
   - 在搜索框中输入公众号名称
   - 点击搜索，系统返回匹配的公众号列表
3. **订阅公众号**：
   - 在搜索结果中找到目标公众号
   - 点击 **"订阅"** 按钮
   - 设置监控间隔（默认 300 秒）
4. **管理订阅**：
   - 查看已订阅的公众号列表
   - 开启/关闭监控
   - 调整监控间隔
   - 取消订阅
5. **查看采集文章**：
   - 点击已订阅公众号，查看该号下采集到的文章列表
   - 支持按时间、互动数据排序

> **前提条件**：使用公众号采集功能需要先在系统中完成微信公众号授权（通过微信扫码登录方式获取授权）。

### 5.3 热门选题

热门选题功能用于多源聚合热点话题，发现优质选题方向。

**操作步骤：**

1. 点击左侧菜单 **🔥 热门选题**
2. **浏览热点**：
   - 页面自动加载当前各平台热点话题
   - 支持按平台筛选（微博、知乎、百度、抖音等）
   - 显示热度值、排名、趋势方向
3. **搜索选题**：
   - 在搜索框中输入关键词
   - 系统通过多搜索引擎聚合搜索
   - 搜索引擎优先级：Tavily → 天工 → MiniMax → 维基百科 → Bing → DuckDuckGo → 百度 → Google
4. **选题评估**：
   - 对感兴趣的话题进行AI评估
   - 评估维度：热度、新颖度、竞争度、匹配度
   - 显示痛点等级和标题模型分析
5. **加入选题库**：
   - 点击 **"加入选题"** 将话题添加到选题库
   - 选题库中的话题可用于后续创作

### 5.4 选题分析

选题分析功能用于深度分析关键词相关内容，发现选题机会。

**操作步骤：**

1. 点击左侧菜单 **📊 选题分析**（在"创作"分组下）
2. **创建分析任务**：
   - 输入要分析的关键词
   - 点击 **"开始分析"**
3. **查看分析结果**：
   - **热门文章**：按点赞数、互动率排序的相关文章
   - **词云分析**：关键词频率可视化
   - **洞察报告**：AI 生成的内容趋势洞察
   - **选题建议**：基于分析的选题推荐
4. **从分析结果创建文章**：
   - 选择感兴趣的选题建议
   - 直接跳转到创作工作台开始创作

### 5.5 创作工作台

创作工作台是核心创作功能，提供6步SOP流程从选题到成文。

**操作步骤：**

1. 点击左侧菜单 **✍️ 创作工作台**（在"创作"分组下）
2. **第1步 - 输入选题**：
   - 选择输入方式：
     - **关键词输入**：直接输入创作关键词
     - **选题库选择**：从已保存的选题中选择
     - **文章参考**：从已采集文章中选择作为参考
   - 点击 **"下一步"**
3. **第2步 - 生成标题**：
   - AI 根据选题生成多个候选标题
   - 可选择一个标题或自行修改
   - 点击 **"下一步"**
4. **第3步 - 生成内容**：
   - AI 根据标题和选题生成文章初稿
   - 可选择写作风格（如已配置）
   - 支持自定义文章长度
   - 点击 **"下一步"**
5. **第4步 - 润色优化**：
   - AI 对文章进行润色和去AI味处理
   - 可在编辑器中手动修改
   - 点击 **"下一步"**
6. **第5步 - 配图**：
   - 系统自动搜索或AI生成配图
   - 选择封面图和正文配图
   - 点击 **"下一步"**
7. **第6步 - 发布**：
   - 选择发布方式：
     - **保存为草稿**：保存到待发布管理
     - **发布到微信**：直接发布到微信公众号（需配置公众号账号）
   - 确认发布

> **重要提示**：创作工作台的所有AI功能依赖系统设置中配置的 LLM 服务商。如未配置 API Key，系统会提示"请先在设置中配置 LLM API Key"。

### 5.6 待发布管理

待发布管理用于管理已创作完成但尚未发布的文章草稿。

**操作步骤：**

1. 点击左侧菜单 **📋 待发布管理**（在"创作"分组下）
2. **查看草稿列表**：
   - 显示所有状态为"草稿"的文章
   - 可按创建时间、标题排序
3. **编辑草稿**：
   - 点击文章进入编辑模式
   - 使用 Markdown 编辑器修改内容
   - 支持实时预览
4. **发布操作**：
   - **发布到微信**：将文章发布到微信公众号草稿箱
   - **直接发布**：发布为正式文章
   - **删除草稿**：移除草稿

### 5.7 数据分析

数据分析功能用于追踪已发布文章的表现数据。

**操作步骤：**

1. 点击左侧菜单 **📊 数据分析**（在"管理"分组下）
2. **文章数据总览**：
   - 总阅读量、总点赞数、总评论数
   - 数据趋势图表
3. **单篇文章分析**：
   - 选择某篇文章查看详细数据
   - 阅读量、点赞数、评论数、分享数的时间变化
   - 互动率分析
4. **数据导出**：
   - 支持导出文章数据为 CSV 格式

### 5.8 闭环优化

闭环优化功能基于已发布文章的数据表现，智能生成优化建议。

**操作步骤：**

1. 点击左侧菜单 **🔄 闭环优化**（在"管理"分组下）
2. **查看优化建议**：
   - 系统自动对比源文章（采集的爆款文章）和改写文章的数据
   - 识别差距类型（如标题吸引力不足、开头不够吸引等）
   - 生成具体的优化建议
3. **应用建议**：
   - 查看每条建议的详细说明
   - 标记建议状态（待处理/已采纳/已忽略）
   - 可将建议应用到 Prompt 模板中
4. **效果追踪**：
   - 追踪已应用建议的效果评分
   - 持续优化创作策略

## 6. 完整工作流示例

以下是一个从选题到发布的完整工作流示例：

### 场景：创作一篇关于"AI编程工具"的公众号文章

**Step 1 - 发现选题**

1. 进入 **🔥 热门选题** 页面
2. 搜索关键词 "AI编程"
3. 浏览搜索结果，发现"Claude Code 实战测评"话题热度较高
4. 点击"选题评估"，AI分析显示该选题热度高、新颖度好
5. 点击"加入选题库"

**Step 2 - 采集素材**

1. 进入 **📥 文章采集** 页面
2. 输入几篇相关微信文章的链接
3. 点击"采集"，文章进入素材库
4. 阅读采集的文章，了解已有内容的写作角度

**Step 3 - 深度分析**

1. 进入 **📊 选题分析** 页面
2. 输入关键词 "Claude Code AI编程"
3. 查看分析结果：
   - 热门文章角度：工具测评、职业影响、实操教程
   - 词云：高频词为"编程"、"AI"、"效率"
   - 洞察：用户关注"实用性"和"学习成本"
4. 确定差异化选题方向："AI编程工具的实际工作流应用"

**Step 4 - AI创作**

1. 进入 **✍️ 创作工作台**
2. 选择"关键词输入"，输入 "Claude Code 实际工作流应用场景"
3. AI生成3个候选标题，选择"用Claude Code重构我的工作流，效率提升300%"
4. AI生成文章初稿，选择"干货实用"风格
5. 在润色步骤中手动调整部分表述
6. AI自动生成配图，选择封面图
7. 保存为草稿

**Step 5 - 审核发布**

1. 进入 **📋 待发布管理**
2. 打开刚创建的草稿，最终审核
3. 确认无误后，点击"发布到微信"
4. 文章发布到微信公众号草稿箱

**Step 6 - 数据追踪**

1. 发布后，进入 **📊 数据分析** 查看阅读数据
2. 进入 **🔄 闭环优化** 查看AI优化建议
3. 根据建议调整后续创作策略

## 7. 常见问题

### Q1: 启动后页面显示 500 错误

**原因**：数据库未启动或数据库连接配置错误。

**解决方案**：
1. 检查 PostgreSQL 是否运行：`pg_isready -h localhost -p 5432`
2. 检查 `.env.local` 中的 `DATABASE_URL` 是否正确
3. 执行数据库 schema 推送：`npx drizzle-kit push`

### Q2: AI 创作功能提示"请先配置 LLM API Key"

**原因**：未在系统设置中配置大模型 API Key。

**解决方案**：
1. 进入 **⚙️ 系统设置 > 🤖 API 配置**
2. 选择服务商并输入 API Key
3. 点击"保存配置"

### Q3: 热门选题搜索无结果

**原因**：搜索 API Key 未配置或已过期。

**解决方案**：
1. 系统会自动回退到免费搜索引擎（维基百科、Bing、DuckDuckGo等）
2. 如需更好的搜索效果，在 `.env.local` 中配置 `TAVILY_API_KEY` 或 `TIANGONG_API_KEY`

### Q4: 微信公众号发布失败

**原因**：公众号 AppID 或 AppSecret 配置错误，或 Access Token 过期。

**解决方案**：
1. 检查 **⚙️ 系统设置 > 📝 公众号配置** 中的 AppID 和 AppSecret
2. 确认公众号已开通相关接口权限
3. 重新保存配置以刷新 Access Token

### Q5: 文章采集失败

**原因**：文章链接格式不正确，或微信反爬限制。

**解决方案**：
1. 确保链接是完整的微信公众号文章 URL（以 `https://mp.weixin.qq.com/` 开头）
2. 如遇反爬限制，稍后重试

### Q6: Docker 方式启动后无法访问

**解决方案**：
1. 检查容器状态：`docker-compose ps`
2. 查看容器日志：`docker-compose logs -f`
3. 确认端口 3003 未被占用：`lsof -i :3003`

### Q7: 数据库 schema 推送失败

**解决方案**：
1. 确认 PostgreSQL 已启动
2. 确认数据库用户有创建表的权限
3. 手动创建数据库：`createdb -U content_monitor content_monitor_db`
4. 重新执行：`npx drizzle-kit push`
